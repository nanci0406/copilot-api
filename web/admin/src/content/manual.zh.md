# Copilot API 管理后台手册

## 首次初始化

1. 在 localhost 上打开 `/admin`。
2. 如果当前还没有 Admin 密钥，请先在 `/admin/setup` 完成一次性初始化。
3. 然后使用保存好的 Admin 密钥在 `/admin/login` 登录后台。

## 安全边界

- Admin 会话认证与网关 API Key 是两套独立机制。
- 在密钥尚未配置前，只有 localhost 可以访问 `/admin/setup`。
- 密钥配置完成后，非 localhost 访问 Admin 必须使用 HTTPS。
- 所有写操作仍由服务端中间件做同源校验。

## 账号管理

- 在 **账号** 页面发起 GitHub 设备码授权流程。
- 切换激活账号时，服务端运行时上下文会立即跟着切换。
- 删除当前激活账号时，服务端会清理或替换对应运行时上下文。

## 模型管理

- Premium multiplier 只影响本地计费统计，不会改变上游真实计费。
- 隐藏模型策略会影响后台其他页面可选的目标模型集合。
- 只有后端声明支持的模型，前端才允许配置 reasoning effort。

## 设置

- 全局限流配置仍然遵守环境变量优先级。
- 上下文管理相关设置仍通过现有配置持久化链路写入。
- 页面中清空 API Key 是显式动作，由服务端负责真正清除。

## 用量与映射

- 用量日志按 cursor 分页，并绑定当前激活账号。
- 模型映射会把客户端暴露的别名转成真实 Copilot 模型 id。
- 如果全局禁用了隐藏模型，映射页面也会同步遵守这条策略。
